# Dream World ‚Äî Domain Glossary

## Utilities

**Disposer** (`src/util/disposer.ts`) ‚Äî collects `THREE.BufferGeometry`, `Material`, `Object3D` instances, unsubscribe functions, and arbitrary cleanup callbacks. Calling `dispose()` runs them all in order. Owned by EntityManager per attached entity ‚Äî entities receive a Disposer from EntityManager via `onAttach(scene, disp)` rather than creating their own.

**PositionTracker** (`src/util/position-tracker.ts`) ‚Äî encapsulates event bus position tracking pattern. Eliminates boilerplate: entities declare what they track (by entityId), tracker handles event subscription, filtering, and Vector3/Quaternion reconstruction. Used by spray and wake entities.

## State

**StateStore** (`src/state/store.ts`) ‚Äî central state management with dotted-path subscriptions. Provides `get(path)`, `set(path, value)`, `subscribe(path, fn)`, `select(selector)`, `watch(selector, fn)`. Pure state primitive ‚Äî no domain logic.

**EntityStateBinding** (`src/state/binding.ts`) ‚Äî encapsulates the subscribe-dispose pattern for entities. Eliminates boilerplate: entities declare what state they need, binding handles subscription mechanics. Used by ocean, lighting, sky, and instance-manager entities.

**LocationTracker** (`src/state/location-tracker.ts`) ‚Äî observes StateStore changes at `environment.*` and `instances.*` paths, marks the active location as dirty. Decouples dirty-tracking logic from the state store. Started by Kernel during init.

## Models

**Model** ‚Äî a 3D object in the world. Has geometry, materials, textures, and a transform. Produced via one of three source types.

**ModelConfig** ‚Äî configuration data that describes a model, defined in `src/model/definitions/<id>/config.ts`. One of `ExtractedModelDef`, `ProceduralModelDef`, or `CompositeModelDef`.

**ModelEntity** ‚Äî the runtime object wrapping a `THREE.Group`. Carries `metadata` (id, source, license, poly count) and lifecycle methods (`dispose()`).

**ModelRegistry** (singleton, `src/model/registry.ts`) ‚Äî central registry of all active models. Created models auto-register; `dispose()` unregisters.

## Source Types

**Extracted model** ‚Äî geometry extracted from an external reference (Poly Haven, etc.) into typed array data files in `.cache/references/<asset>/data/`. Compiled to GLB at build time.

**Procedural model** ‚Äî geometry generated by a TypeScript function in `src/generators/`. Compiled to GLB at build time.

**Composite model** ‚Äî geometry assembled from primitives (`src/primitives/`) + sub-model references. Compiled to GLB at build time.

## Materials

**MaterialSpec** (`src/material/types.ts`) ‚Äî canonical description of a material. Currently unused by runtime ‚Äî materials are baked into GLB. Candidate for deletion.

## Time

**WorldClock** (singleton, `src/util/world-clock.ts`) ‚Äî monotonically increasing elapsed time accumulated from `dt`. Entities read `worldClock.elapsed` instead of managing local `t` counters. Provides `timeScale` and `paused`.

**EntityManager** (singleton, `src/entity/manager.ts`) ‚Äî owns the entity list, provides `attach/detach/update`. Does NOT own the RAF loop (contradicts ADR-002). Update is driven by `simulationPlugin` which calls `entityManager.update(dt)` inside the Kernel's render loop, only in 'play' mode.

**EntityRegistry** (`src/entity/entity-registry.ts`) ‚Äî singleton registry of `EntityFactory` adapters. `register(factory)` adds a factory; `createAll(config, modelLoader, store?)` runs all registered factories and merges results. Used by WorldLoader to produce `SceneEntity[]` from a `WorldConfig`.

**BehaviorRegistry** (`src/entity/behavior-registry.ts`) ‚Äî singleton registry of `BehaviorFactory` adapters keyed by string. `register(name, factory)` adds a factory; `create(id, def, deps)` dispatches to the factory for the instance's `behavior` field. Instance dispatch in `register-factories.ts` iterates `config.instances`, looks up `def.behavior` in the registry, and delegates. No hardcoded behavior strings in dispatch code.

**BehaviorFactory** ‚Äî interface: `create(id: string, def: InstanceDef, deps: { modelLoader, store }): Promise<SceneEntity[]>`. Registered per behavior name. The vessel behavior factory (in `src/entity/vessel/ship.ts`) creates the vessel + spray + wake group. A static placement factory is unnecessary ‚Äî static instances are handled at runtime by the instance-manager entity. Adding a new interactive object type: one file, one `behaviorRegistry.register()` call.

## Identity caching (scene gate)

**ISceneObject.id** ‚Äî stable domain identity string on every scene object wrapper. Set once at construction (delegates to the wrapped `THREE.Object3D.uuid`). Used by the scene gate's identity cache to guarantee wrapper referential identity across calls.

**SceneAdapter identity cache** ‚Äî two Maps in `SceneAdapter`: `idCache` (id ‚Üí wrapper) and `vendorCache` (Object3D ‚Üí id). `getObjectByName()` and `traverse()` consult the cache before constructing a new `SceneObject`, guaranteeing wrapper referential identity. Evicted on `remove()`. The single `(obj as any).object3D` cast happens inside `SceneAdapter.add()` and is the gate's private implementation detail ‚Äî never at a plugin or application seam. Note: `SceneObject.children` and `parent` accessors still create ephemeral wrappers (not yet cached).

**EntityManager** (singleton) ‚Äî owns the entity list, lifecycle control. RAF loop is in Kernel. (ADR-002 ‚Äî note: RAF loop was delegated to Kernel for edit/play mode switching; ADR-002's "owns the RAF loop" is stale.)

**Event bus** (singleton, `src/util/event-bus.ts`) ‚Äî decouples entities. Events: `entity:attached`, `entity:detached`, `entity:position-changed`.

## World

**WorldConfig** (singular, unified from `LocationPreset` + deprecated `WorldConfig`) ‚Äî declarative scene composition. One shape used by WorldLoader and Store. Contains:
  - `environment` ‚Äî sky, waves, ocean, lighting, fog state
  - `instances` ‚Äî model placement with per-instance `behavior` field

**Behavior** ‚Äî string field on `InstanceDef` in a WorldConfig. Used as a dispatch key by `BehaviorRegistry` at world-load time. Built-in values: `'vessel'` (wave-response physics + spray + wake). Any other string triggers the registered `BehaviorFactory` or a warning. Default `undefined` means static placement handled by instance-manager at runtime. Fully extensible ‚Äî no hardcoded dispatch logic.

**WorldLoader** (`src/model/world-loader.ts`) ‚Äî loads a WorldConfig, orchestrates entity creation via `EntityRegistry.createAll()`. Returns `WorldLoadResult`. Pure orchestration ‚Äî delegates all construction to registered factory adapters. Imports `src/entity/register-factories.ts` for side-effect auto-registration.

**EntityFactory** ‚Äî interface in `src/entity/entity-registry.ts`: `match(config, modelLoader, store?): Promise<FactoryResult>`. Each factory inspects the config and produces entities. Environment entities (ocean, sky, lighting) `match` on config presence; instance entities (vessel) `match` on behavior field. Each entity module registers its own factory at module level ‚Äî no central wiring file.

**Kernel** ‚Äî bootstrap orchestrator. Thin orchestrator that delegates to specialized modules: RenderingModule (Three.js), PluginManager (plugin lifecycle), StateStore (app state), LocationTracker (dirty tracking). `setMode(m)` propagates to EntityManager (`setPaused`) and plugins (`onModeSwitch`).

**RenderingModule** (`src/graphics/module.ts`) ‚Äî owns the Three.js scene, renderer, camera, and OrbitControls. Manages window resize events and the RAF render loop. Accepts an `onBeforeRender` callback for pre-render updates (plugin rendering, entity updates).

**PluginManager** (`src/plugins/plugin-manager.ts`) ‚Äî owns plugin lifecycle (register, init, onModeSwitch, render). Registry inlined ‚Äî no separate PluginRegistry module. Isolates plugin crashes during mode switching.

**PluginStateAPI** (`src/plugins/plugin-state-api.ts`) ‚Äî adapter providing plugins a stable interface to application state (get, set, select, watch, subscribe). Decouples plugins from direct StateStore implementation.

**PluginSceneAPI** (`src/plugins/plugin-scene-api.ts`) ‚Äî adapter providing plugins a stable interface to the 3D scene (add, remove, getObjectByName, traverse). Accepts and returns `ISceneObject` (never vendor types). Delegates every call to `IScene` verbatim.

**Plugin-level escape hatch** ‚Äî when a plugin genuinely needs Three.js interop that the gate cannot reasonably wrap (TransformControls, raycaster internals), the vendor exposure is contained to that single plugin module rather than leaking through a gate interface. Used by gizmos plugin only. Exception to Rule 4, documented as deliberate carve-out.

## Gate interfaces (deepened)

**IMaterial** ‚Äî material abstraction. No `.raw` escape hatch. Provides `dispose()`. Created via `createSkyMaterial()`, `createRingMaterial()`, `createWaterMaterial()` in `src/graphics/materials.ts`. Internally stores `_vendor` for the gate's `createMesh` to access (cast inside the gate only).

**IRenderer** ‚Äî renderer abstraction. No `.raw`. Provides `domElement: HTMLElement`, `info`, `dispose()`.

**ICamera** ‚Äî camera abstraction. No `.raw`. Provides `aspect: number`, `updateProjectionMatrix()`. The `RenderingModule` getter attaches a `_vendorCam` property (not on the interface) as a plugin-level escape hatch for TransformControls interop in the gizmos plugin.

**IScene.createMesh** ‚Äî factory method on the scene gate: `createMesh(geometry: BufferGeometry, material: IMaterial): ISceneObject`. Entity code passes inline geometry and an IMaterial; the gate creates the vendor Mesh internally. No vendor types cross the seam.

**EntityManager** ‚Äî owns the entity list and per-entity Disposers. `setPaused(bool)` controls whether `update(dt)` advances entity state. Paused in 'edit' mode, unpaused in 'play' mode.

## Waves

**`sampleOcean(x, z, t)`** (`src/environment/waves.ts`) ‚Äî returns `{ height, dispX, dispZ }`. Gerstner wave simulation.

**`sampleNormal(x, z, t)`** ‚Äî returns `{ x, y, z }`. Wave surface normal.

## Asset Pipeline (ADR-003)

The asset pipeline transforms model configs into compiled GLB artifacts. Two stages:

### Pipeline Stages

- **compile** (`scripts/pipeline/compile.ts`) ‚Äî reads `src/model/definitions/<id>/config.ts` for each model. For extracted models, reads geometry from `.cache/references/<asset>/data/`. For procedural models, imports the generator function and runs it. Outputs `public/models/<id>.glb`.
- **publish** (`scripts/pipeline/publish.ts`) ‚Äî scans `public/models/` for GLBs, reads metadata from configs, writes `public/models/manifest.json`.

Run via: `npm run setup`

### Reference Downloads

External asset sources are downloaded via a standalone tool, not part of the pipeline:

- **`npm run reference`** ‚Äî downloads assets from Poly Haven etc. to `.cache/references/<asset>/`. Implementations in `scripts/reference/providers/`.

### Model Creation Paths

Three ways to create a model, defined in `src/model/definitions/<id>/config.ts`:

1. **extracted** ‚Äî references an external asset in `.cache/references/`
2. **procedural** ‚Äî references a generator function in `src/generators/`
3. **composite** ‚Äî assembles from primitives + sub-model refs

### Primitives

**Primitives** (`src/primitives/`) ‚Äî library of `buildBox()`, `buildCylinder()`, `buildSphere()`, `buildPlane()`, `buildLathe()` that return `THREE.BufferGeometry`.

### Generators

**Generators** (`src/generators/`) ‚Äî parameterized functions: `generateHull()`, `generateBuoy()`, `generateIsland()`. Called at compile time, results baked into GLB.

### Runtime Loaders

**Loaders** (`src/model/`) ‚Äî runtime modules for loading compiled artifacts:

- **GlbLoader** ‚Äî wraps Three.js GLTFLoader, returns `THREE.Group`
- **ModelLoader** ‚Äî resolves model refs via catalog, loads GLBs, applies overrides, caches `ModelEntity` instances
- **Catalog** ‚Äî reads `public/models/manifest.json`, provides `getEntry(ref)`
- **WorldLoader** ‚Äî see World section above
- **WorldLoadResult / WorldLoadError** ‚Äî types defined in `src/model/types.ts` alongside the loaders that produce them (moved from the deleted `src/worlds/` directory)
